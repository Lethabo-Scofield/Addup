Build a modern Stripe-style landing page for Addup.

Addup is a financial data reliability layer for businesses. It helps finance teams prevent bad financial data, automatically match transactions, resolve reconciliation exceptions, and verify results with clear explanations.

Goal:
Create a premium, modern landing page with a waitlist form.

Tech stack:
- Next.js
- TypeScript
- Tailwind CSS
- Framer Motion
- SQLite or simple local JSON storage for waitlist emails

Design direction:
- Stripe-inspired structure
- Clean white base
- Soft gradients
- Premium spacing
- Rounded cards
- Smooth motion
- Modular product storytelling
- No old “Hero / Features / Pricing” generic layout
- Mobile responsive

Main message:
“Close your books in minutes, not days.”

Subtext:
“Addup cleans, matches, and verifies financial data so finance teams review decisions instead of chasing transactions.”

Page structure:

1. Hero
- Large headline: “Close your books in minutes, not days.”
- Subtext
- CTA: “Join the waitlist”
- Secondary CTA: “See how it works”
- Right-side animated system visual:
  Bank data → Cleaned data → Matched transactions → Exceptions → Verified books

2. Trust strip
Short credibility line:
“Built for finance teams that need cleaner books, faster closes, and fewer reconciliation surprises.”

Add small labels:
Bank feeds
Accounting systems
Invoices
Audit logs

3. Product narrative blocks
Each block must have text on one side and an animated visual on the other.

Block 1: Prevent bad data
Copy:
“Addup catches missing references, duplicate entries, inconsistent vendor names, and incomplete records before they break reconciliation.”
Visual:
Messy transaction cards transforming into clean structured records.

Block 2: Match intelligently
Copy:
“Addup groups related payments, invoices, and bank entries using confidence-based matching, not rigid rules only.”
Visual:
Transaction cards connecting with confidence scores like 94%, 88%, 76%.

Block 3: Resolve only what matters
Copy:
“Instead of reviewing hundreds of transactions, teams review a short decision queue.”
Visual:
A queue of unresolved exceptions with actions: Accept, Split, Link, Investigate.

Block 4: Explain and verify
Copy:
“Every match includes a reason, source trail, and audit-ready explanation.”
Visual:
A reasoning panel showing:
Matched because amount, date proximity, vendor pattern, and reference similarity aligned.

4. Interactive “How Addup works” section
Use tabs or cards:
- Capture
- Clean
- Match
- Resolve
- Verify

Each tab should update a small mock product UI.

5. Use cases section
Cards for:
- Finance teams
- Accountants
- SMEs
- Multi-entity businesses

Each card should explain the pain in one sentence and the Addup outcome in one sentence.

6. Waitlist section
Headline:
“Get early access to Addup.”

Fields:
- Email
- Company name optional
- Role optional

Button:
“Join waitlist”

Behavior:
- Validate email
- Prevent duplicate emails
- Show success message: “You’re on the list.”
- Show error message for invalid or duplicate email

7. Footer
Olyxee
Research and Infrastructure for Artificial Intelligence
Links:
Privacy
Contact

Backend/API:
Create POST /api/waitlist

Request body:
{
  "email": "person@company.com",
  "company": "Company Name",
  "role": "Finance Manager"
}

Validation:
- email required
- valid email format
- company optional
- role optional
- duplicate emails not allowed

Storage:
Use SQLite if easy. Otherwise use a local JSON file for MVP.

UI quality requirements:
- No placeholder lorem ipsum
- No broken links
- No console errors
- Fully responsive
- Smooth but lightweight animations
- Clear component structure
- Use reusable components
- Keep copy short and sharp
- Make it feel like a real fintech infrastructure product, not a template

Deliverables:
- Complete working Next.js app
- Landing page
- Waitlist API
- Basic storage
- Clear instructions in README for running locally

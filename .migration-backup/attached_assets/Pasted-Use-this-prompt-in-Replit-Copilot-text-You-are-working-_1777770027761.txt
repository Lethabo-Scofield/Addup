3. ADD FUZZY MATCHING ENGINE
==================================================

Replace exact matching with candidate ranking.

For every valid bank transaction, compare against valid ledger transactions.

Calculate:

amount_score:
- exact same signed amount: 1.0
- same absolute amount but opposite sign: 0.75 with warning "direction differs"
- percentage difference <= 2%: 0.9
- percentage difference <= 5%: 0.8
- percentage difference <= 10%: 0.65
- percentage difference <= 15%: 0.5
- else: 0.0

date_score:
- same date: 1.0
- 1 day difference: 0.95
- 2–3 days difference: 0.85
- 4–7 days difference: 0.65
- 8–14 days difference: 0.35
- else: 0.0

description_score:
Use normalized descriptions.
- exact normalized description match: 1.0
- synonym match: 0.95
- strong fuzzy similarity: 0.8
- moderate fuzzy similarity: 0.6
- weak similarity: 0.35
- no similarity: 0.0

Use a string similarity function such as:
- token similarity
- Levenshtein ratio
- Jaro-Winkler
- built-in simple implementation if no package exists

final_score:
final_score =
  0.45 * amount_score +
  0.30 * description_score +
  0.25 * date_score

Important:
Do not match if amount_score is 0 unless description_score >= 0.95 and date_score >= 0.85.
Do not match invalid rows.
Do not auto-match if there are multiple similar candidates with close scores.

Manual Review:
- final_score >= 0.40 and < 0.55
- or duplicate-like candidates exist
- or direction differs
- or amount difference > 10%
- or date difference > 7 days

Unmatched Bank:
- no candidate above 0.40

Unmatched Ledger:
- valid ledger transaction not used in any match or possible match

Invalid:
